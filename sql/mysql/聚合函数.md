聚合函数（Aggregate Functions）是 SQL 中用于对一组数据进行计算并返回单一结果的函数，常用于统计分析、数据汇总等场景。它们通常与 `GROUP BY` 子句配合使用，也可单独用于对全表数据进行计算。


### **一、常用聚合函数**
#### 1. `COUNT()`：统计记录数量
用于计算表中符合条件的记录行数或非 NULL 值的数量。

- **语法**：
  ```sql
  COUNT(expr)  -- 统计expr不为NULL的记录数
  COUNT(*)     -- 统计所有记录（包括NULL）
  COUNT(DISTINCT expr)  -- 统计expr去重后的非NULL值数量
  ```

- **示例**：
  ```sql
  -- 统计users表总人数
  SELECT COUNT COUNT(*) AS total_users FROM users;

  -- 统计有邮箱的用户数（排除email为NULL的记录）
  SELECT COUNT(email) AS email_users FROM users;

  -- 统计不同年龄段的人数（去重）
  SELECT COUNT(DISTINCT age) AS distinct_ages FROM users;
  ```


#### 2. `SUM()`：计算总和
用于计算数值列的总和，忽略 NULL 值。

- **语法**：
  ```sql
  SUM(expr)  -- 计算expr的总和
  SUM(DISTINCT expr)  -- 计算expr去重后的总和
  ```

- **示例**：
  ```sql
  -- 计算所有订单的总金额
  SELECT SUM(amount) AS total_amount FROM orders;

  -- 计算不同商品的总销量（去重后）
  SELECT SUM(DISTINCT quantity) AS unique_quantity FROM order_details;
  ```

- **注意**：`SUM()` 仅对数值类型有效，对字符串或日期类型会返回 `0` 或错误。


#### 3. `AVG()`：计算平均值
用于计算数值列的平均值，忽略 NULL 值（计算公式：总和/非NULL值数量）。

- **语法**：
  ```sql
  AVG(expr)  -- 计算expr的平均值
  AVG(DISTINCT expr)  -- 计算expr去重后的平均值
  ```

- **示例**：
  ```sql
  -- 计算用户的平均年龄
  SELECT AVG(age) AS avg_age FROM users;

  -- 计算不同商品的平均单价（去重后）
  SELECT AVG(DISTINCT price) AS avg_unique_price FROM products;
  ```


#### 4. `MAX()`：获取最大值
返回指定列中的最大值，适用于数值、字符串、日期等类型。

- **语法**：
  ```sql
  MAX(expr)  -- 返回expr的最大值
  ```

- **示例**：
  ```sql
  -- 获取最大年龄
  SELECT MAX(age) AS max_age FROM users;

  -- 获取最新的订单日期
  SELECT MAX(order_date) AS latest_order FROM orders;

  -- 获取字典序最大的用户名（字符串按ASCII码比较）
  SELECT MAX(username) AS max_username FROM users;
  ```


#### 5. `MIN()`：获取最小值
返回指定列中的最小值，适用类型与 `MAX()` 相同。

- **语法**：
  ```sql
  MIN(expr)  -- 返回expr的最小值
  ```

- **示例**：
  ```sql
  -- 获取最小年龄
  SELECT MIN(age) AS min_age FROM users;

  -- 获取最早的注册日期
  SELECT MIN(created_at) AS earliest_reg FROM users;
  ```


### **二、聚合函数与 `GROUP BY` 配合使用**
聚合函数默认对全表数据进行计算，而 `GROUP BY` 可以将数据按指定列分组，然后对每组分别应用聚合函数。

#### 1. 基础用法
```sql
-- 按性别分组，统计每组人数和平均年龄
SELECT
  gender,
  COUNT(*) AS count,  -- 每组人数
  AVG(age) AS avg_age  -- 每组平均年龄
FROM users
GROUP BY gender;  -- 按gender分组
```

#### 2. 多列分组
```sql
-- 按性别和城市分组，统计每组人数
SELECT
  gender,
  city,
  COUNT(*) AS count
FROM users
GROUP BY gender, city;  -- 先按gender分，再按city分
```

#### 3. `GROUP BY` + `HAVING`：过滤分组结果
`WHERE` 用于分组前过滤行，`HAVING` 用于分组后过滤分组结果（可直接使用聚合函数结果）。

```sql
-- 统计人数超过100的城市及其平均年龄
SELECT
  city,
  COUNT(*) AS population,
  AVG(age) AS avg_age
FROM users
GROUP BY city
HAVING population > 100;  -- 过滤分组后人数>100的城市
```


### **三、注意事项**
1. **NULL 值处理**：
   - 所有聚合函数均忽略 NULL 值（`COUNT(*)` 除外，它统计所有行，包括 NULL）。
   - 示例：`AVG(age)` 计算时会排除 `age` 为 NULL 的记录。

2. **与非聚合列的配合**：
   - 当使用 `GROUP BY` 时，`SELECT` 后只能出现 **分组列** 或 **聚合函数**，否则结果无意义（不同数据库对这点的严格性不同，MySQL 非严格模式下可能允许，但不推荐）。
   - 错误示例：
     ```sql
     -- 错误：username不是分组列，也不是聚合函数
     SELECT gender, username, COUNT(*) FROM users GROUP BY gender;
     ```

3. **`DISTINCT` 与聚合函数**：
   - `DISTINCT` 可用于聚合函数内部，对计算对象去重后再聚合（如 `COUNT(DISTINCT age)`）。
   - 注意：`DISTINCT` 后面只能跟一个列（某些数据库支持多列，如 MySQL）。

4. **性能考量**：
   - 聚合函数对大表计算可能较慢，建议为分组列或聚合列创建索引。
   - 避免在 `HAVING` 中使用复杂计算，可先用 `WHERE` 过滤无关数据。


### **四、高级聚合场景**
#### 1. 嵌套聚合函数
聚合函数可以嵌套使用（外层聚合基于内层聚合的结果）：
```sql
-- 计算各城市平均年龄的最大值
SELECT MAX(avg_age) AS max_city_avg_age
FROM (
  SELECT city, AVG(age) AS avg_age
  FROM users
  GROUP BY city
) AS city_ages;  -- 子查询先计算每个城市的平均年龄
```

#### 2. 条件聚合（`CASE WHEN` + 聚合函数）
通过 `CASE WHEN` 在聚合时添加条件过滤：
```sql
-- 分别统计男性和女性的人数
SELECT
  COUNT(CASE WHEN gender = 'male' THEN 1 END) AS male_count,
  COUNT(CASE WHEN gender = 'female' THEN 1 END) AS female_count
FROM users;
```
- 原理：`CASE WHEN` 满足条件时返回 `1`，否则返回 `NULL`，`COUNT()` 只统计非 NULL 值。


### **总结**
聚合函数是 SQL 数据分析的核心工具，掌握 `COUNT()`、`SUM()`、`AVG()`、`MAX()`、`MIN()` 的用法，结合 `GROUP BY` 和 `HAVING` 可以实现复杂的统计需求。实际使用时需注意 NULL 值处理、分组逻辑及性能优化，确保结果准确且高效。
