# MySQL数据更新与综合查询详解（含建库建表+数据插入）
## 一、基础环境搭建（建库、建表、插入测试数据）
先创建测试用数据库、数据表，并插入基础数据，后续所有更新与查询操作均基于此环境。

### 1. 创建数据库
```sql
-- 创建数据库（判断是否存在，不存在则创建）
CREATE DATABASE IF NOT EXISTS student_manage DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 使用该数据库
USE student_manage;
```

### 2. 创建数据表（学生表、成绩表、课程表）
#### （1）学生表（student）
存储学生基础信息
```sql
CREATE TABLE IF NOT EXISTS student (
  stu_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '学生ID（主键，自增）',
  stu_name VARCHAR(20) NOT NULL COMMENT '学生姓名',
  gender CHAR(2) NOT NULL COMMENT '性别（男/女）',
  age INT NOT NULL COMMENT '年龄',
  class VARCHAR(20) NOT NULL COMMENT '班级',
  phone VARCHAR(11) COMMENT '手机号（可为空）',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间（默认当前时间）'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT '学生信息表';
```

#### （2）课程表（course）
存储课程信息
```sql
CREATE TABLE IF NOT EXISTS course (
  course_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '课程ID（主键，自增）',
  course_name VARCHAR(50) NOT NULL COMMENT '课程名称',
  teacher VARCHAR(20) NOT NULL COMMENT '授课教师',
  credit INT NOT NULL COMMENT '学分'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT '课程信息表';
```

#### （3）成绩表（score）
存储学生课程成绩（关联学生表和课程表）
```sql
CREATE TABLE IF NOT EXISTS score (
  score_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '成绩记录ID（主键，自增）',
  stu_id INT NOT NULL COMMENT '学生ID（关联student表的stu_id）',
  course_id INT NOT NULL COMMENT '课程ID（关联course表的course_id）',
  score INT NOT NULL COMMENT '分数（0-100）',
  exam_time DATE NOT NULL COMMENT '考试时间',
  -- 外键约束：确保stu_id和course_id在关联表中存在
  FOREIGN KEY (stu_id) REFERENCES student(stu_id) ON DELETE CASCADE,
  FOREIGN KEY (course_id) REFERENCES course(course_id) ON DELETE CASCADE,
  -- 联合唯一约束：一个学生一门课程只能有一条成绩记录
  UNIQUE KEY uk_stu_course (stu_id, course_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT '学生成绩表';
```

## 二、数据更新操作（UPDATE）
用于修改表中已存在的数据，核心语法：  
`UPDATE 表名 SET 字段1=值1, 字段2=值2,... WHERE 条件;`  
⚠️ 注意：**无WHERE条件时会更新表中所有数据，请务必谨慎使用！**

### 1. 基础更新（单表、单字段/多字段）
#### （1）更新单字段
例：将学生“孙七”（stu_id=5）的年龄改为19
```sql
UPDATE student 
SET age = 19 
WHERE stu_id = 5; -- 条件：stu_id=5（精准定位）
```

#### （2）更新多字段
例：将学生“赵六”（stu_id=4）的手机号改为13800138009，班级改为“高一(3)班”
```sql
UPDATE student 
SET phone = '13800138009', class = '高一(3)班' 
WHERE stu_id = 4;
```

### 2. 带条件的更新（WHERE子句进阶）
#### （1）范围条件（>、<、>=、<=、BETWEEN...AND）
例：将“高一(1)班”中年龄小于18的学生年龄改为18
```sql
UPDATE student 
SET age = 18 
WHERE class = '高一(1)班' AND age < 18;
```

例：将数学（course_id=2）成绩在70~80分之间的学生分数提高5分
```sql
UPDATE score 
SET score = score + 5 
WHERE course_id = 2 AND score BETWEEN 70 AND 80;
```

#### （2）模糊条件（LIKE）
例：将姓名以“张”开头的学生手机号改为13800138010
```sql
UPDATE student 
SET phone = '13800138010' 
WHERE stu_name LIKE '张%'; -- %表示任意字符（0个或多个）
```

#### （3）IN条件（匹配多个值）
例：将学生ID为1、3、5的英语（course_id=3）成绩改为原分数的95%（保留整数）
```sql
UPDATE score 
SET score = FLOOR(score * 0.95) -- FLOOR()向下取整
WHERE stu_id IN (1,3,5) AND course_id = 3;
```

### 3. 关联表更新（多表联合更新）
语法：`UPDATE 表1 JOIN 表2 ON 关联条件 SET 字段=值 WHERE 筛选条件;`  
例：将“王老师”授课的课程成绩低于80分的学生分数提高3分
```sql
-- 关联score表和course表，通过course_id匹配
UPDATE score s
JOIN course c ON s.course_id = c.course_id 
SET s.score = s.score + 3 
WHERE c.teacher = '王老师' AND s.score < 80;
```


## 三、综合查询操作（SELECT进阶）
综合查询是MySQL核心能力，结合`JOIN`（关联）、`WHERE`（筛选）、`GROUP BY`（分组）、`HAVING`（分组筛选）、`ORDER BY`（排序）、`LIMIT`（分页）等子句实现复杂数据提取。

### 1. 基础查询（复习）
```sql
-- 查询学生表所有数据
SELECT * FROM student;

-- 查询学生姓名、性别、班级（指定字段）
SELECT stu_name, gender, class FROM student;

-- 别名（AS，简化显示）
SELECT stu_name AS 姓名, gender AS 性别, class AS 班级 FROM student;
```

### 2. 条件筛选查询（WHERE）
例：查询“高一(1)班”性别为男的学生姓名和年龄
```sql
SELECT stu_name, age 
FROM student 
WHERE class = '高一(1)班' AND gender = '男';
```

例：查询英语成绩大于90分的学生ID和分数
```sql
SELECT stu_id, score 
FROM score 
WHERE course_id = 3 AND score > 90;
```

### 3. 关联表查询（JOIN，核心重点）
用于从多个关联表中提取数据，常用`INNER JOIN`（内连接）、`LEFT JOIN`（左连接）、`RIGHT JOIN`（右连接）。

#### （1）内连接（INNER JOIN）：只显示两表中匹配的数据
例：查询学生姓名、课程名称、分数（关联3张表）
```sql
SELECT 
  s.stu_name AS 学生姓名,
  c.course_name AS 课程名称,
  sc.score AS 分数
FROM student s -- s是student表别名
INNER JOIN score sc ON s.stu_id = sc.stu_id -- 关联学生表和成绩表
INNER JOIN course c ON sc.course_id = c.course_id; -- 关联成绩表和课程表
```

#### （2）左连接（LEFT JOIN）：显示左表所有数据，右表匹配不到则为NULL
例：查询所有学生的姓名、语文成绩（即使未考语文，也显示学生信息）
```sql
SELECT 
  s.stu_name AS 学生姓名,
  sc.score AS 语文成绩
FROM student s
LEFT JOIN score sc ON s.stu_id = sc.stu_id 
LEFT JOIN course c ON sc.course_id = c.course_id 
WHERE c.course_name = '语文'; -- 筛选语文课程
```

#### （3）右连接（RIGHT JOIN）：显示右表所有数据，左表匹配不到则为NULL
例：查询所有课程的名称、授课教师，以及选择该课程的学生姓名（即使无学生选，也显示课程）
```sql
SELECT 
  c.course_name AS 课程名称,
  c.teacher AS 授课教师,
  s.stu_name AS 学生姓名
FROM student s
RIGHT JOIN score sc ON s.stu_id = sc.stu_id
RIGHT JOIN course c ON sc.course_id = c.course_id;
```

### 4. 分组查询（GROUP BY + 聚合函数）
聚合函数：`COUNT()`（计数）、`SUM()`（求和）、`AVG()`（平均值）、`MAX()`（最大值）、`MIN()`（最小值）  
`GROUP BY`：按指定字段分组，聚合函数作用于每组数据。

#### （1）按单字段分组
例：按班级分组，查询每个班级的学生人数
```sql
SELECT 
  class AS 班级,
  COUNT(stu_id) AS 学生人数 -- COUNT(stu_id)统计每组非NULL的stu_id数量
FROM student 
GROUP BY class;
```

例：按课程分组，查询每门课程的平均分、最高分、最低分
```sql
SELECT 
  c.course_name AS 课程名称,
  ROUND(AVG(sc.score), 1) AS 平均分, -- ROUND(...,1)保留1位小数
  MAX(sc.score) AS 最高分,
  MIN(sc.score) AS 最低分
FROM course c
INNER JOIN score sc ON c.course_id = sc.course_id 
GROUP BY c.course_id, c.course_name; -- 分组字段需包含非聚合字段
```

#### （2）按多字段分组
例：按班级和性别分组，查询每个班级不同性别的学生人数
```sql
SELECT 
  class AS 班级,
  gender AS 性别,
  COUNT(stu_id) AS 人数
FROM student 
GROUP BY class, gender; -- 先按班级分组，再按性别分组
```

#### （3）分组筛选（HAVING）
`WHERE`筛选行数据，`HAVING`筛选分组后的结果（需配合`GROUP BY`使用）  
例：查询平均分大于85分的课程名称和平均分
```sql
SELECT 
  c.course_name AS 课程名称,
  AVG(sc.score) AS 平均分
FROM course c
INNER JOIN score sc ON c.course_id = sc.course_id 
GROUP BY c.course_id, c.course_name
HAVING AVG(sc.score) > 85; -- 筛选平均分>85的分组
```

### 5. 排序查询（ORDER BY）
语法：`ORDER BY 字段1 [ASC/DESC], 字段2 [ASC/DESC]`  
- `ASC`：升序（默认，可省略）  
- `DESC`：降序  

例：查询学生的姓名、数学成绩，按成绩降序排列（分数高的在前）
```sql
SELECT 
  s.stu_name AS 学生姓名,
  sc.score AS 数学成绩
FROM student s
INNER JOIN score sc ON s.stu_id = sc.stu_id 
INNER JOIN course c ON sc.course_id = c.course_id 
WHERE c.course_name = '数学'
ORDER BY sc.score DESC;
```

例：查询所有成绩记录，先按课程ID升序，再按分数降序
```sql
SELECT * FROM score 
ORDER BY course_id ASC, score DESC;
```

### 6. 分页查询（LIMIT）
语法：`LIMIT 起始索引, 每页条数`（起始索引从0开始）  
用于实现分页显示（如网页分页，每页显示10条数据）

例：查询学生表，每页显示2条，查询第2页（即第3、4条数据）
```sql
-- 第1页：LIMIT 0,2（第1-2条）
-- 第2页：LIMIT 2,2（第3-4条）
-- 第n页：LIMIT (n-1)*2,2
SELECT * FROM student 
LIMIT 2, 2;
```

例：查询语文成绩前3名的学生姓名和分数（按分数降序，取前3条）
```sql
SELECT 
  s.stu_name AS 学生姓名,
  sc.score AS 语文成绩
FROM student s
INNER JOIN score sc ON s.stu_id = sc.stu_id 
INNER JOIN course c ON sc.course_id = c.course_id 
WHERE c.course_name = '语文'
ORDER BY sc.score DESC
LIMIT 3; -- 取前3条（等价于LIMIT 0,3）
```

### 7. 复杂综合查询案例
例：查询“高一(1)班”学生的姓名、所选课程名称、分数、授课教师，要求分数大于80分，按课程名称升序、分数降序排列，每页显示2条，查询第1页
```sql
SELECT 
  s.stu_name AS 学生姓名,
  c.course_name AS 课程名称,
  sc.score AS 分数,
  c.teacher AS 授课教师
FROM student s
INNER JOIN score sc ON s.stu_id = sc.stu_id 
INNER JOIN course c ON sc.course_id = c.course_id 
-- 筛选条件：班级为高一(1)班，分数>80
WHERE s.class = '高一(1)班' AND sc.score > 80
-- 排序：先按课程名称升序，再按分数降序
ORDER BY c.course_name ASC, sc.score DESC
-- 分页：第1页，每页2条
LIMIT 2;
```


## 四、注意事项
1. **更新操作**：无WHERE条件会全表更新，建议先执行SELECT查询验证条件是否正确，再执行UPDATE。  
2. **外键约束**：更新关联表时，需确保外键字段值在关联表中存在，否则会报错。  
3. **分组查询**：GROUP BY后SELECT子句中只能包含分组字段和聚合函数，否则会出现数据混乱（MySQL8.0以上默认严格模式）。  
4. **分页索引**：LIMIT起始索引从0开始，而非1，分页公式需准确（第n页：LIMIT (n-1)*每页条数, 每页条数）。
